# QuietMind Development Document

## 1. 核心设计哲学 (Design Philosophy)
本项目不是传统的番茄钟，而是基于“前额叶渐进式超载”和“无损级联”概念的认知训练模型。
强调：
*   **真实有效专注**：剥离所有无意义的耗时。
*   **高颗粒度事件流**：精确打断惩罚机制。
*   **状态继承**：当日或历次的疲劳度必须影响下一次的目标设定。

## 2. 软件架构 (Architecture)
项目严格遵守 MVVM (Model-View-ViewModel) 架构，并强化了 Domain 层的计算纯度：
*   **UI Layer (Jetpack Compose)**: 仅负责视图渲染和事件转发。
*   **Presentation (ViewModel)**: 处理沙盒流程（时间线管理），调用数据仓储获取历史数据，利用 Domain 计算下一目标。
*   **Domain (FocusAlgorithms)**: 单纯的 Kotlin 数学函数，严禁混入 Android / Room 相关 API。
*   **Data (Room + KSP)**: 持久化每次专注特征值。

## 3. 五大统计算法模块图解 (Core Algorithms)
所有的系统运转逻辑被高度内聚在 `FocusAlgorithms.kt` 单例对象中，保障纯函数式推演。以下是核心统计算法的变量定义、计算原理与公式说明：

### 3.1 专注纯度得分 (FQS - Focus Quality Score)
**原理**：评估单场专注的心流密度。系统鼓励**超额执行**（实际专注时间超越目标设定），且对运行过程中的任何“打断走神”施加指数级的惩罚。通过在超载领域引入**对数阻尼（Logarithmic Dampening）**，防止极端心流状态下的数值指数爆炸。

*   **变量说明**：
    *   $T_{actual}$ = 实际执行时长（分钟）
    *   $T_{target}$ = 目标设定时长（分钟）
    *   $N$ = 期间打断的总次数
    *   $\rho$ = 打断密度 ( $N / T_{actual}$ )
    *   $k$ = 惩罚指数常量（系统默认置为 `2.0`）
*   **计算公式**：
    1. 计算完成率系数 $R$：
       若 $T_{actual} \le T_{target}$：$R = \frac{T_{actual}}{T_{target}}$
       若 $T_{actual} > T_{target}$（触发对数阻尼）：$R = 1.0 + \ln(1.0 + \frac{T_{actual} - T_{target}}{T_{target}})$
    2. 计算纯净乘数 $P$：$P = e^{-k \cdot \rho}$
    3. **最终产出**：$FQS = R \times 100 \times P$

### 3.2 有效脑力做功 (EFD - Effective Focus Duration)
**原理**：剔除磨洋工带来的假象，测算大脑前额叶在该任务下实际付出的心智总工作量。它是推测下一天认知容量（BaseTarget）的绝对货币单位。

*   **变量说明**：
    *   $T_{actual}$ = 实际执行时长（分钟）
    *   $FQS$ = 前一项产出的专注纯度得分
    *   $W$ = 任务难度系数（例如硬核逻辑 2.5，深度阅读 1.5，日常事务 1.0）
*   **计算公式**：
    *   **最终产出**：$EFD = T_{actual} \times \frac{FQS}{100} \times W$

### 3.3 抗干扰半衰期 (DRI - Distraction Resistance Index)
**原理**：测算大脑在绝对无干扰情况下“破防”的临界点时间，评估长时间深潜的心如止水定力。这通常是基于一天的不同场次聚合计算的。

*   **变量说明**：
    *   $L_{first\_distraction}$ = 当日每场专注中，**第一次**走神发生的时间点集合 $t_1, t_2, ...$
*   **计算公式**：
    *   **最终产出**：$DRI = \frac{\sum{L}}{L\text{的长度}}$ (也就是当日首次打断时间的均值)

### 3.4 动态目标推导体系 (Target Auto-Scaling & CRF)
**原理**：游戏核心——“前额叶渐进式超载”。基于历史 EFD （代表长期的体力底盘）和近日 EFD （代表近期的干劲），来预测接下来的合理目标。并附带**防雪崩衰减**与**超额跳跃限制**。

*   **(A) 日基础目标 (Daily Base Target)**：新的一天的起步目标推演。
    *   引入历史加权做功 $E_{weighted} = (0.4 \times EFD_{总均值}) + (0.6 \times EFD_{近7日均值})$
    *   **公式转换**：将做功除以默认难度常数转回绝对时长 $T_{raw} = \frac{E_{weighted}}{2.5}$
    *   **双向对数平滑 (Bi-directional Logarithmic Dampening)**：若 $T_{raw}$ 偏离了默认起步基准（例如 60 分钟），针对偏移差值启用对数收敛：
        *   **超载抑制**（向上跃进时的阻力）：$T_{dampened} = 60 + 20 \times \ln(1.0 + \frac{T_{raw} - 60}{20})$，防止目标无限制膨胀。
        *   **防坠控制 / Soft Floor**（向下跌落时的安全网）：应对连续状态低迷。例如算力认为只需 30 分钟，系统不会直接腰斩，而是计算亏空差值并进行缓冲扣除：$T_{dampened} = 60 - 20 \times \ln(1.0 + \frac{60 - T_{raw}}{20})$。这缓和了从 60 分钟骤降到 15 分钟的陡峭落差。
    *   最后，此目标会被全局强制截断在 `[15分钟, 180分钟]` 绝对安全区间之内。
*   **(B) 日内认知次级联 (CRF - Target_N)**：同一天内做完第一场后，推演下一场目标的增减比例。依据前一场的 $FQS$ 提供弹性乘数：
    *   $FQS \ge 100$（心流延续）：系统保底增长 5% (`1.05`)，之后每超额 1%，系统额外附加 0.5% 的目标增长，单次跳跃封顶 `1.30`（至多单次+30%）。
    *   $90 \le FQS < 100$（平稳消耗）：保持不长不短，乘数 `1.00`。
    *   $75 \le FQS < 90$（略微降落）：微观缩水，乘数 `0.90`。
    *   $60 \le FQS < 75$（显著疲劳）：削减目标力求休息恢复，乘数 `0.80`。
    *   $FQS < 60$（严重透支）：心智溃败防线，紧急释放压力，乘数 `0.65`。

### 3.5 全局神经重塑评级 (GNPS - Global Neuroplasticity Score)
**原理**：整合多维历史反馈，用于主页面呈现“游戏段位（RPG化等级分）”。全分段设计倾向于总满分 1000 左右。

*   **组成模块与权重分布**：
    1.  **耐力 (Endurance) [40%，满分 400分]**：评估当前与基石周的比率 $E = (\frac{EFD_{近7天平}}{EFD_{首周平}}) \times 100$，限高计算。
    2.  **纯净度 (Purity) [30%，满分 300分]**：评估历史全量打断密度的干净程度 $P = (1.0 - \rho_{总均值}) \times 300$。
    3.  **抗压定力 (Resistance) [30%，满分 300分]**：将抵抗走神长达的分钟数换算为积分 $R = DRI_{总均值} \times 10$。
*   **计算公式**：将每个分项的当前获得分数计算出比例，并乘以千分制预设总权重：
    $GNPS = (\frac{E}{400} \times 1000 \times 0.4) + (\frac{P}{300} \times 1000 \times 0.3) + (\frac{R}{300} \times 1000 \times 0.3)$
